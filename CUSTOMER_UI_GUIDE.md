## Color Scheme

### Item Type Colors
- **Vegetarian**: `bg-green-100 text-green-600`
- **Non-Vegetarian**: `bg-red-100 text-red-600`
- **Vegan**: `bg-emerald-100 text-emerald-600`
- **Egg**: `bg-amber-100 text-amber-600`

### Badge Colors
- **Bestseller**: `bg-amber-500` (gold)
- **Spicy**: `bg-red-500` (red)
- **Restaurant Type - Veg**: `bg-green-600` (green)
- **Restaurant Type - Non-Veg**: `bg-red-600` (red)
- **Restaurant Type - Both**: `bg-amber-600` (amber)

### Interactive Elements
- **Primary Button**: Primary theme color
- **Outline Button**: Border with transparent background
- **Active Tab**: Primary theme color with solid background
- **Inactive Tab**: Muted with transparent background

## Responsive Behavior

### Mobile (< 1280px)
- Single column layout for menu items
- Smaller images (w-24)
- Compact text sizes
- Bottom sheet for filters
- Touch-friendly tap targets

### Desktop (â‰¥ 1280px)
- Two column layout for menu items
- Larger images (w-40)
- Larger text sizes
- More spacing between elements
- Hover effects on cards

---

## User Interactions

### Click/Tap Actions
1. **Item Card**: Opens item details dialog
2. **Favorite Button**: Toggles favorite status
3. **Add Button**: Adds item to cart
4. **Info Button**: Opens item details dialog
5. **Filter Button**: Opens filters sheet
6. **Tab**: Filters items by type
7. **Image Dot**: Changes carousel image

### Visual Feedback
- **Hover**: Card shadow increases, border color changes
- **Active Filter**: Highlighted with primary color
- **Loading**: Spinner animation
- **Success**: Toast notification

---

## Accessibility Features

- **Icons with Labels**: All icons have accompanying text
- **Color + Icon**: Not relying on color alone
- **Touch Targets**: Minimum 44x44px on mobile
- **Keyboard Navigation**: Tab through interactive elements
- **Screen Reader**: Semantic HTML and ARIA labels

---

## Performance Optimizations

- **Lazy Loading**: Images load as needed
- **Responsive Images**: Different sizes for mobile/desktop
- **Efficient Filtering**: Client-side filtering without API calls
- **Smooth Animations**: CSS transitions for better performance
- **Optimized Re-renders**: React memo and useMemo where needed
